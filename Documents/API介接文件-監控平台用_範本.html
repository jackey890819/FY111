上傳流程

開始操作時，請先更新操作紀錄，再上傳成績&職災&關卡，有更新操作紀錄後，就可以進行成績&職災&關卡的單一測試及更新了!!

再次更新操作紀錄，則為新的操作紀錄，即之後上傳的成績&職災&關卡皆屬於該操作紀錄!!

更新操作紀錄

[POST] /api/operationRecord

Request

headers

Accept: application/json
Content-Type: application/json
apiToken: XXXXXXXXXXXXXXXXXXXXXXX(監控人員，監控平台登入驗證成功時回傳的token)
stsp: E84?FFqYm!kfAX%H(監控平台專屬金鑰)
JID: 11105230001(操作者工號)

body

{
    "training_id": 47,
    "unit_code": "SF_U"
}

comments

{
    "training_id": "(受訓代號)",
    "unit_code": "(大單元代碼)"
